## Run this at the start of any new Cowork project folder to bootstrap CLAUDE.md files

I want you to write my CLAUDE.MD files - instructions every future session will follow.
First, look through this folder: the files, the subfolders, how everything is named and organized, and any existing CLAUDE.md files.
Then interview me: ask up to 8 questions, one at a time, about how I like to work, what I keep having to correct or repeat, what my deliverables need to look like, and anything you can't out from my files.

Then write up to three files:

* A CLAUDE.md in my main Cowork folder: only things true for everything I make - my tone, my formats, my rules for how you work with me. 
* A CLAUDE.md in this projects folder: only things specific to this project - what it is, who it's for, where things live, what the finished output looks like.
* if any subfolder genuinely works differently (say, one is client-facing copy), a short CLAUDE.md just for that folder

Rules for writing them: be specific, never vague ("start every draft with the hook, not background", not "write engagingly").

Keep each file well under 200 lines - shorter is better. Include only instructions I'd otherwise have to repeat in every chat - nothing you can see from my files anyway. Never repeat a rule across two files. Where a rule isn't obvious, add its one-line reason. Every line has to earn it's place. 

Show me each file before saving it, and tell me what you deliberately left out and why.
